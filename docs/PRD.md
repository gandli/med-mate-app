# 药易懂 — 产品需求文档 (PRD)

> 让每一颗药都被患者真正理解

## 1. 产品概述

### 1.1 产品定位
面向患者及家属的智能用药管理工具，解决"医生开了药但看不懂、记不住、吃不对"的核心痛点。

### 1.2 一句话描述
拍一张处方，AI 帮你翻译成人话，告诉你怎么吃、什么时候吃、要注意什么。

### 1.3 产品形态
微信小程序（优先） + 后续考虑独立 App

### 1.4 目标用户

| 用户角色 | 典型画像 | 核心需求 |
|---------|---------|---------|
| **主要用户：照顾者** | 25-45岁子女，父母有慢性病多药服用 | 帮父母管药、远程监督吃药 |
| **直接用户：患者** | 45-75岁慢病患者，长期服药 | 知道药是干嘛的、什么时候吃 |
| **次要用户：年轻患者** | 20-40岁，偶尔看病开药 | 快速了解药品信息、查相互作用 |

## 2. 痛点·爽点·痒点

### 2.1 😣 痛点（恐惧驱动，不解决会出问题）

| 痛点 | 恐惧来源 | 解决方案 |
|------|---------|---------|
| 不知道药之间会不会"打架" | 怕吃出事、怕中毒 | 药物相互作用自动检测，录入即扫描，红色警告不可忽略 |
| 药吃错了/漏吃了不知道后果 | 怕病情加重、怕再住院 | 智能提醒 + 漏服处理建议 |
| 老人独居/子女不在身边 | 子女的愧疚和担心 | 家属协同看板，老人点"已服药"，子女实时可见 |
| 多种药吃法不同搞混了 | 怕吃错剂量/时间 | 今日药卡，每个时段该吃什么、吃几粒一目了然 |
| 中西药一起吃不知道要不要隔开 | 怕影响药效或出副作用 | 中西药间隔提醒，自动在时间表中错开 |
| 肝肾不好还在吃伤肝肾的药 | 怕雪上加霜 | 肝肾负担评估，多药叠加风险可视化 |

### 2.2 😆 爽点（即时满足，用完就爽）

| 爽点 | 爽在哪 | 解决方案 |
|------|--------|---------|
| 拿到一袋药完全懵 → 拍一张照3秒全懂 | 焦虑到掌控的瞬间翻转 | 处方拍照 → AI秒出人话解读 |
| 说明书看不懂 → 一句话告诉你这药干嘛的 | "原来这个是保护心脏的" | 一句话功效 + 重要程度标签 |
| 十几种药记不住 → 一张今日药卡搞定 | 不用想，照着吃就行 | 每日用药时间表，按餐次自动归类 |
| 去医院复查 → 一键导出用药报告 | 医生一看就懂 | PDF用药报告 |
| 药快吃完了 → 自动提醒续药 | 不慌，不会断药 | 库存倒计时 + 续药清单 |

### 2.3 🤩 痒点（理想自我，不急但想要）

| 痒点 | 理想自我 | 解决方案 |
|------|---------|---------|
| "想做个靠谱的子女" | 远程也能照顾好父母 | 家属端：帮建药单、看记录、收漏服提醒 |
| "想搞懂自己的身体" | 对健康有掌控感 | 用药档案：历史记录、就诊时间线、变更追踪 |
| "想问但不好意思问医生" | 打破信息不对称 | AI问答 |
| "想科学养生不交智商税" | 分清药和保健品 | 保健品vs药品标识，录入时自动区分 |
| "朋友推荐的药能不能吃" | 有个专业参谋 | 自购药风险检查：扫码检测与处方冲突 |
| "中药方子看着好高深" | 想懂但不好意思说不懂 | 中药方解读："经典XX方加减，调理XX" |

### 2.4 三层产品逻辑

```
痒点（拉新）→ "拍一下就全懂了，推荐给爸妈用"
  ↓
爽点（激活）→ 首次拍照识别的Aha moment
  ↓
痛点（留存）→ 每天提醒+安全检测+家属协同，离不开了
```

## 3. 核心问题

### 2.1 用户痛点

1. **信息不对称** — 医生问诊时间短，没时间逐一解释每种药的作用
2. **说明书反人类** — 专业术语密集、字小、信息过载，老年人根本无法阅读
3. **多药管理混乱** — 慢病患者常服 5-16 种药，吃药时间各不相同（餐前/随餐/餐后/睡前），极易漏服、错服
4. **隐性风险不可见** — 药物相互作用、肝肾损伤风险、特殊人群禁忌，患者完全不知道
5. **依从性差** — 记不住→漏吃→病情反复→再住院，形成恶性循环
6. **药品来源多样** — 不只是医院处方：院内制剂（医院自制药，没有国药准字）、药店自购药（OTC/处方外流）信息更不透明，患者更难判断

### 2.2 现有方案不足

| 现有方案 | 问题 |
|---------|------|
| 药品说明书 | 不说人话，老年人看不了 |
| 问医生/药师 | 时间有限，问完就忘 |
| 百度搜索 | 信息杂乱，广告多，容易误导 |
| 用药助手（丁香园） | 面向医生，专业门槛高 |
| 手机闹钟提醒 | 只能提醒时间，不知道吃什么 |

### 3.2 P0：药品识别录入

#### 录入方式

| 方式 | 实现难度 | 准确度 | 用户场景 |
|------|---------|-------|---------|
| **处方单拍照 OCR** | 中 | 高 | 拿到医院打印的处方/出院医嘱，拍照批量录入 |
| **药盒条形码扫描** | 低 | 最高 | 一盒一盒扫，适合在家整理药品 |
| **药盒拍照 OCR** | 中 | 中高 | 对着药盒拍照识别药名和规格 |
| **药品追溯码扫描** | 低 | 最高 | 扫码即得完整药品信息 |
| **手动搜索** | 低 | — | 兜底方案，搜索药品名添加 |
| **药店小票拍照** | 中 | 中高 | 药店购药后拍小票批量录入 |
| **院内制剂手动录入** | 低 | — | 院内制剂无国药准字，扫码/数据库难匹配，需手动输入药名+拍照，AI辅助解读 |
| **药片外观识别** | 高 | 中 | 药片散落分不清时拍照识别（后期） |

#### MVP 策略
优先实现：**处方单拍照 + 条形码扫描 + 手动搜索**

#### 药品来源场景

| 来源 | 特点 | 产品应对 |
|------|------|---------|
| **医院处方药** | 有处方单、有国药准字、信息最完整 | OCR识别 + 数据库匹配，主流场景 |
| **院内制剂** | 医院自制，无国药准字号（只有院内制剂批号），外部数据库查不到，包装简陋信息少 | 拍药瓶/标签OCR + AI解读 + 标注"院内制剂，建议咨询开药医院" |
| **药店自购（OTC）** | 有条形码和国药准字，但没有医嘱用法 | 扫码录入 + AI根据说明书生成用法建议 + 标注"自购药，如与处方药同服请咨询药师" |
| **药店处方外流** | 持医院处方去药店买，有处方但可能买替代品牌 | 支持手动修改药品品牌但保留处方用法 |
| **网购药品** | 电商平台购买，有订单截图 | 后期支持订单截图识别（P3） |
| **中成药** | 有国药准字，成分为中药配方，药店/医院均有 | 正常扫码/OCR录入，AI解读侧重"功效主治+体质适用+饮食禁忌" |
| **中药饮片（汤剂）** | 医生开方抓药，无条形码，手写/打印处方，涉及君臣佐使配伍 | 处方拍照OCR + AI解读整方功效 + 煎煮方法 + 服用时间 + 忌口 |
| **中药配方颗粒** | 单味颗粒冲服替代煎煮，有包装可扫码 | 扫码录入 + 识别组方 + AI解读同汤剂 |
| **中药外用/膏方/丸散** | 院内制剂或药店成品，剂型多样 | 拍照/手动录入 + 标注用法（外敷/口服/冲服） |

#### 处方识别流程
```
拍照/选图 → OCR 文字提取 → AI 结构化解析 → 药品列表确认 → 入库
```
- 支持医院打印处方、手写处方（手写识别难度高，标注"识别不确定"让用户确认）
- 识别内容：药品名称、规格、用法用量、频次
- 用户确认环节不可省略，避免识别错误导致用药风险

### 3.3 P0："说人话"药品解读

这是产品**核心差异化能力**。不是复读说明书，而是翻译成普通人能懂的语言。

#### 每种药品展示以下信息卡片：

**基础信息**
- 药品名（通用名 + 商品名）
- 一句话说清作用 — 例："保护心脏血管，防止血栓形成"
- 重要程度标签 — 🔴 关键药物（不可停） 🟡 重要辅助 🟢 对症缓解

**用法说明（人话版）**
- 什么时候吃 — "每天早上饭后吃1片"
- 怎么吃 — "整片吞服，不要嚼碎" / "需要含在舌下"
- 吃多久 — "长期服用" / "吃7天停药"

**注意事项（分级展示）**
- 🔴 严重警告 — "绝对不能和XX药一起吃"
- 🟡 重要提醒 — "吃这个药期间不能喝酒"、"不能吃柚子/西柚"
- 🟢 一般提示 — "可能会有点头晕，属于正常反应"

**特殊人群标注（醒目图标）**
- 🤰 孕妇禁用/慎用
- 🤱 哺乳期禁用/慎用
- 👶 儿童禁用/剂量调整
- 🧓 老年人注意事项
- 🫘 肝功能不全注意
- 🫁 肾功能不全注意

**副作用（通俗版）**
- 不说"肝毒性"，说"如果尿液变深黄色、眼睛发黄，立刻去医院"
- 不说"低血压"，说"起床别太猛，慢慢站起来"
- 区分"常见但不严重"和"少见但要警惕"

**中药/中成药专属解读**
- 整方功效 — "这个方子主要是补气养血，适合体虚乏力的人"
- 体质适用 — "适合怕冷、手脚凉的人；上火体质慎用"
- 忌口 — "吃药期间忌生冷、辛辣、绿豆、萝卜"
- 煎煮指导（中药饮片）— 先煎/后下/包煎/另煎、煎煮时间、用水量
- 服用温度 — "趁热喝" / "放温喝" / "凉服"
- 中西药间隔 — "中药和西药间隔至少30分钟-1小时服用"

### 3.4 P0：每日用药时间表

将所有药品按服药时间自动归类，生成一目了然的每日计划。

#### 时间段划分
| 时间段 | 说明 | 示例时间 |
|-------|------|---------|
| 🌅 晨起空腹 | 起床后立即，早餐前30分钟以上 | 06:30 |
| 🍳 早餐前 | 早餐前15-30分钟 | 07:00 |
| 🥣 早餐时 | 随餐服用 | 07:30 |
| 🍳 早餐后 | 餐后15-30分钟 | 08:00 |
| 🍚 午餐前 | 午餐前15-30分钟 | 11:30 |
| 🍜 午餐时 | 随餐服用 | 12:00 |
| 🍚 午餐后 | 餐后15-30分钟 | 12:30 |
| 🍲 晚餐前 | 晚餐前15-30分钟 | 17:30 |
| 🥘 晚餐时 | 随餐服用 | 18:00 |
| 🍲 晚餐后 | 餐后15-30分钟 | 18:30 |
| 🌙 睡前 | 睡前30分钟 | 21:30 |

#### 展示形式
- **今日药卡** — 一张卡片展示当前/下一个时间段该吃的药，大字体，老年人友好
- **完整时间表** — 全天所有时间段的药品列表
- 每个药品可打 ✅ 标记已服用
- 支持自定义各餐时间（适应不同作息）

### 3.5 P1：用药提醒

- 基于每日时间表自动生成提醒
- 微信服务通知推送（小程序订阅消息）
- 提醒内容包含：时间段 + 药品名称 + 数量
- 未确认服药的，间隔15分钟二次提醒
- 支持关闭/调整某个时段的提醒

---

### 3.6 P1：药物相互作用检测

#### 检测维度
| 维度 | 说明 | 展示 |
|------|------|------|
| 药物-药物相互作用 | 两种药之间的已知交互 | 🔴危险 🟡注意 🟢安全 |
| 药物-食物相互作用 | 常见食物禁忌（酒精、柚子、牛奶等） | 饮食提醒卡片 |
| 重复用药检测 | 不同商品名但同一成分 | ⚠️ 重复警告 |
| 肝肾负担评估 | 多药叠加对肝肾的综合压力 | 负担等级：低/中/高 |
| **中西药同服风险** | 中成药/中药与西药之间的已知交互（如含麻黄的中成药+降压药、含甘草的方剂+强心苷类） | 🔴🟡 分级提醒 |
| **中药配伍禁忌** | 十八反、十九畏等传统配伍禁忌 | ⚠️ 配伍警告 |
| **中成药成分重叠** | 多种中成药含相同成分导致过量（如多药含附子/朱砂） | ⚠️ 成分超量警告 |

#### 交互设计
- 添加新药时自动触发检测
- 发现 🔴 级别风险时弹窗警告，建议咨询医生
- 所有检测结果可在"安全报告"页面汇总查看

### 3.7 P2：家属协同

- 子女扫码绑定家庭成员
- 远程查看父母今日用药完成情况
- 父母漏服药时子女收到提醒
- 子女可帮父母录入药品、调整提醒时间
- 支持多位家属绑定同一位患者

---

### 3.8 P2：AI 用药问答

基于用户当前药品清单的上下文智能问答：
- "今天早上忘吃了，要不要现在补？"
- "我感冒了能不能自己买感冒药吃？"
- "吃这个药能喝咖啡吗？"
- "最近总觉得头晕是不是药的副作用？"

**安全红线**：
- 所有回答附加"以上仅供参考，具体请咨询医生"
- 涉及停药/换药/加药一律引导就医
- 不做诊断，不推荐用药

---

### 3.9 P3：扩展功能

#### 药品库存管理
- 录入时填写数量
- 根据用量自动计算剩余天数
- 不足7天提醒续药
- 一键生成"需续药清单"方便去医院/药店

#### 复查日历
- 录入医生嘱咐的复查时间
- 到期前3天/1天提醒
- 关联复查项目（验血/心电图/影像等）

#### 药品过期提醒
- 拍照识别有效期
- 到期前30天提醒
- 过期药品标红警告

#### 家庭药箱
- 多成员药品独立管理
- 切换查看不同家庭成员的用药方案
- 家庭用药安全报告

## 5. 技术架构

### 4.1 技术选型

| 层级 | 方案 | 说明 |
|------|------|------|
| 前端 | 微信小程序（原生/uni-app） | 覆盖面最广，老年人无需安装 |
| 后端 | Supabase / 轻量云函数 | BaaS优先，降低运维成本 |
| OCR | 微信OCR API / 百度OCR / PaddleOCR | 处方识别 |
| 条码识别 | wx.scanCode | 微信原生能力 |
| AI 解读 | 大模型 API（DeepSeek/Qwen/Kimi） | 药品信息翻译+问答 |
| 药品数据库 | 国家药监局公开数据 + 自建知识库 | 药品基础信息 |
| 相互作用数据 | DrugBank / 自建规则库 | 药物交互检测 |

### 4.2 AI 解读引擎

核心 Prompt 策略：
- 输入：药品名称 + 规格 + 用法 + 患者基本信息（年龄/性别/肝肾功能）
- 输出：结构化的"人话版"药品解读卡片
- 预生成常用药的解读缓存，减少 API 调用
- 冷门药实时生成 + 人工审核后入库

### 4.3 数据安全
- 用药信息属于个人健康隐私，需加密存储
- 遵守《个人信息保护法》和《数据安全法》
- 用户可随时删除自己的数据
- 不向第三方提供用户用药数据

## 6. 合规与风险

### 5.1 法律合规

| 风险点 | 应对策略 |
|-------|---------|
| 不能提供诊断建议 | 全程定位为"信息整理工具"，不做诊断 |
| 不能推荐用药 | 只解读已开处方药，不推荐新药 |
| AI 解读可能有误 | 附加免责声明 + 建立审核机制 |
| 健康数据隐私 | 加密存储 + 用户可删除 + 不对外共享 |
| 医疗器械/软件资质 | 定位为健康管理工具，非医疗器械；需法律咨询确认边界 |

### 5.2 产品风险
- **识别准确率** — OCR 识别错误可能导致用药错误 → 强制用户确认环节
- **AI 幻觉** — 大模型可能生成不准确的药品信息 → 核心药品信息预审核入库
- **用户过度依赖** — 用户可能以此替代医嘱 → 明确引导"有疑问请咨询医生"

- 药店合作：续药引流

- 保险公司：慢病管理数据
